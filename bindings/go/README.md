# attested-key-zk Go bindings

Self-contained Go module for the attested-key-zk approval proof circuit.

```sh
go get github.com/naughtbot/attested-key-zk/bindings/go
```

No tarball download, no environment variables, no build script — cgo compiles
the vendored C++ source on first build (~5–10 s) and caches the result.

## Build prerequisites

- A C++17 compiler in `PATH` (`clang++` or `g++`).
- OpenSSL development headers:
  - **Linux:** `apt-get install libssl-dev` (or distro equivalent).
  - **macOS arm64:** `brew install openssl@3`.

## Behavior under `CGO_ENABLED=0`

The package compiles using stub implementations that return
`attestedkeyzk.ErrUnavailable` from `GenerateCircuit`, `Prove`, `Verify`, and
`CircuitID`. This lets pure-Go binaries (e.g. minimal-Docker builds) keep
compiling while making the missing capability explicit at call time.

## Provenance

The vendored C++ sources are a snapshot copy of upstream
[`google/longfellow-zk`](https://github.com/google/longfellow-zk) plus this
repository's own `src/`. See [`PROVENANCE.md`](PROVENANCE.md) for the exact
upstream commit and the upstream-to-vendored file mapping. The snapshot is
regenerated by `scripts/sync-go-bindings.sh` in the parent repo whenever the
submodule is bumped.
